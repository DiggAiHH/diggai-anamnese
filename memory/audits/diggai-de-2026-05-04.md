# diggai.de — Live-Audit 2026-05-04

**Auditor:** claude-code (opus-4-7) · **Tool:** web_fetch · **Scope:** UX/Conversion + DSGVO

---

## P0 — KRITISCH (sofort)

### F-01 · API-Subdomain SSL-Cert ungültig
- **Befund:** `https://api-takios.diggai.de` → `ERR_CERT_COMMON_NAME_INVALID`
- **Auswirkung:** Frontend macht `<link rel="preconnect" href="https://api-takios.diggai.de" crossorigin />` — alle API-Calls scheitern beim Endnutzer. App ist effektiv tot, sobald der erste Auth/Patient-Call rausgeht.
- **Fix:** Netlify-DNS / Cert für `api-takios.diggai.de` erneuern oder CNAME korrekt aufsetzen. Verifizieren mit `openssl s_client -connect api-takios.diggai.de:443 -servername api-takios.diggai.de`.

### F-02 · DSGVO: Google Fonts werden direkt von Google geladen
- **Befund:** `<link href="https://fonts.googleapis.com/css2?family=Inter…&family=Noto+Sans+Arabic…">` + preconnect zu `fonts.gstatic.com`
- **Auswirkung:** IP des Patienten geht an Google (USA). Mehrere DSGVO-Urteile in DE (LG München I 3 O 17493/20) sehen das als rechtswidrig.
- **Fix:** Inter + Noto Sans Arabic selbst hosten via `@fontsource/inter` und `@fontsource/noto-sans-arabic` aus `node_modules`. Preconnect-Tags raus.

## P1 — HOCH

### F-03 · og:url zeigt auf Netlify-Subdomain statt diggai.de
- **Befund:** `<meta property="og:url" content="https://diggai-drklaproth.netlify.app">` — falsche URL für Social-Sharing.
- **Fix:** Auf `https://diggai.de` ändern in `index.html`.

### F-04 · favicon ist Vite-Default `/vite.svg`
- **Befund:** Kein eigenes Brand-Icon. `apple-touch-icon` zeigt auf `/icons/icon-192.svg` (gut), aber `<link rel="icon">` hängt am Vite-Default.
- **Fix:** `<link rel="icon" type="image/svg+xml" href="/icons/icon-192.svg">` (oder dediziertes favicon.svg).

### F-05 · `<title>` route-unabhängig immer "Anamnese-Fragebogen"
- **Befund:** /, /datenschutz, /impressum, /admin etc. liefern alle SPA-Fallback mit identischem Title.
- **Auswirkung:** SEO + Browser-Tab UX. Patienten sehen 5 gleichnamige Tabs.
- **Fix:** `react-helmet-async` oder route-spezifisches `<title>`-Setting im Router. Idealerweise SSR/Prerender für statische Routes.

### F-06 · Kein robots.txt, kein sitemap.xml
- **Befund:** `/robots.txt` und `/sitemap.xml` liefern HTML (SPA-Fallback) statt echte Crawler-Files.
- **Fix:** `public/robots.txt` und `public/sitemap.xml` anlegen, Netlify-`_redirects` so konfigurieren dass diese Pfade vor SPA-Catchall ausgeliefert werden.

## P2 — MITTEL

### F-07 · og:image fehlt komplett
- **Befund:** Keine `og:image` und `og:image:width/height` Meta-Tags.
- **Auswirkung:** Geteilte Links in WhatsApp/Slack/LinkedIn zeigen nur grauen Platzhalter.
- **Fix:** 1200×630 PNG mit Logo+Claim, in `public/og-image.png`, `<meta property="og:image" content="https://diggai.de/og-image.png">`.

### F-08 · Theme-Color `#3b82f6` ist Tailwind-Default
- **Befund:** PWA-`theme_color` und `<meta name="theme-color">` nutzen Tailwind blue-500, nicht das DiggAi-Brand-Blau.
- **Fix:** Auf Brand-Hex aus `branding.json` mappen.

### F-09 · `orientation: portrait-primary` im Manifest zwingt Mobile in Portrait
- **Befund:** Manifest setzt feste Orientierung. Tablets/Mobile in Landscape werden gedreht — schlechte UX bei der Anamnese-Eingabe auf iPad.
- **Fix:** Auf `"orientation": "any"` oder ganz weglassen.
